# 错误码手册

本文档列出企业业务系统常见错误码，内容为模拟资料。

## 支付与退款错误码

### E1001 用户账号被冻结

含义：用户账号处于冻结状态，无法继续提交退款、售后或敏感操作。

排查步骤：

1. 使用用户账号状态查询接口确认用户状态。
2. 如果 status 为 frozen，查看 freeze_reason 字段。
3. 如果 risk_level 为 high，需要转人工风控复核。
4. 如果冻结原因为登录异常，建议用户完成身份验证后再重试。

### E2001 支付渠道临时不可用

含义：支付渠道网络抖动、服务维护或上游响应超时。

处理建议：

- 等待 5 至 10 分钟后重试退款。
- 如果连续失败 3 次，需要记录渠道流水号并转交支付值班人员。

### E3001 订单金额与支付流水不一致

含义：订单应退金额与支付渠道流水金额不一致，系统拒绝自动退款。

处理建议：

- 核对订单金额、优惠券抵扣和实际支付金额。
- 需要人工复核后再决定是否发起退款。
