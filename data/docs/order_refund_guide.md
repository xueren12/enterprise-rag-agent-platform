# 订单退款流程

本文档用于说明企业内部订单退款的标准处理流程，内容为演示数据，不包含真实公司信息。

## 适用范围

- 用户已完成支付，但需要取消订单或申请售后退款。
- 订单状态为已支付、待发货、已发货、售后处理中时，需要按不同规则处理。
- 虚拟商品、风控订单和跨境订单需要人工复核。

## 标准操作步骤

1. 客服在订单中心根据订单号查询订单状态、支付渠道和售后记录。
2. 如果订单未发货，客服可提交退款申请，系统进入自动审核。
3. 如果订单已发货，客服需要先确认退货物流签收状态，再提交退款申请。
4. 财务系统根据支付渠道发起原路退款，退款单状态变为 processing。
5. 支付渠道返回成功后，退款单状态更新为 success，并通过消息中心通知用户。
6. 如果超过 30 分钟仍为 processing，需要查询退款状态并触发退款状态同步任务。

## 退款失败处理

- 如果退款失败并出现错误码 E2001，通常表示支付渠道临时不可用，应稍后重试。
- 如果退款失败并出现错误码 E3001，通常表示订单金额与支付流水不一致，需要人工复核。
- 如果用户反馈未到账，但系统显示 success，应先核对支付渠道流水号，再建议用户查看原支付账户。

## 相关业务工具

- 查询订单退款状态：sql_query_tool，入参为 order_id。
- 触发退款状态同步任务：script_task_tool，入参为 task_name=refund_status_sync 且 confirm=true。
